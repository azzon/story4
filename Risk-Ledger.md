## 一、合规风险 · 平台规则与内容禁忌

> **整理专家**: PM (项目经理) + QA (质检编辑)
> **参考来源**: 起点中文网作者后台、番茄小说创作规范、七猫作者指南、实际封禁案例

### 1.1 政治敏感内容红线

| 编号 | 禁忌内容 | 风险等级 | 来源 |
|------|----------|----------|------|
| P-01 | **历史虚无主义**：否定或丑化革命领袖、英雄人物、重大历史事件 | 🔴 封禁 | 起点官方规则 |
| P-02 | **政治讽刺**：影射、暗喻、丑化现任或前任国家领导人及政府机构 | 🔴 封禁 | 全平台通用 |
| P-03 | **敏感历史时期**：涉及文革、六四等敏感历史事件的直接描写 | 🔴 封禁 | 全平台通用 |
| P-04 | **分裂国家**：涉及港独、台独、藏独、疆独等分裂势力的正面描写 | 🔴 封禁+法律风险 | 全平台通用 |
| P-05 | **军队负面形象**：丑化中国军队、抹黑军人形象 | 🔴 封禁 | 起点官方规则 |

**安全操作指南**:
- 架空历史世界观时，避免使用与现实朝代高度相似的名称
- 如需涉及历史，遵循"正史正说"原则
- 军人角色必须正面或中性，禁止负面塑造

### 1.2 色情暴力边界

| 编号 | 禁忌内容 | 风险等级 | 来源 |
|------|----------|----------|------|
| S-01 | **露骨性描写**：直接描写性器官、性行为细节 | 🔴 封禁 | 全平台通用 |
| S-02 | **暗示性描写**：擦边球式的性暗示（可用省略号、场景切换替代） | 🟡 警告/锁章 | 起点审核实践 |
| S-03 | **未成年人色情**：任何涉及未成年人的性暗示或色情描写 | 🔴 封禁+法律风险 | 全平台通用 |
| S-04 | **血腥暴力细节**：过度详细的肢解、酷刑、内脏描写 | 🟡 锁章 | 起点官方规则 |
| S-05 | **虐待描写**：详细的虐待过程（人或动物） | 🟡 锁章 | 番茄官方规则 |
| S-06 | **自残自杀**：详细描写自残、自杀的方式和过程 | 🔴 封禁 | 全平台通用 |

**安全操作指南**:
- 情感戏份止于"房门关上"，后续留白
- 战斗场面聚焦动作和力量，而非伤口细节
- 血腥场面用"感官冲击"代替"视觉堆砌"（如：铁锈味、黏腻感）

### 1.3 宗教、民族、迷信禁忌

| 编号 | 禁忌内容 | 风险等级 | 来源 |
|------|----------|----------|------|
| R-01 | **宗教诋毁**：侮辱、丑化任何宗教（佛教、道教、伊斯兰教等） | 🔴 封禁 | 全平台通用 |
| R-02 | **宗教冲突**：煽动不同宗教信仰之间的对立和仇恨 | 🔴 封禁 | 全平台通用 |
| R-03 | **民族歧视**：对任何民族的贬低、歧视性描写 | 🔴 封禁+法律风险 | 全平台通用 |
| R-04 | **封建迷信宣扬**：将算命、占卜、风水等作为"真实有效"的手段正面宣传 | 🟡 警告 | 番茄官方规则 |
| R-05 | **邪教宣传**：涉及邪教组织的正面描写或宣传其思想 | 🔴 封禁+法律风险 | 全平台通用 |

**安全操作指南**:
- 涉及宗教元素时，保持客观中立的态度
- 算命/风水等可以作为"江湖骗术"或"神秘学"出现，但不能暗示其真实有效
- 避免将任何民族与负面特征绑定

## 五、熔断机制 (S01-QA-038)

> **责任专家**: QA (质检编辑) + PM (项目经理)
> **任务ID**: S01-QA-038
> **完成日期**: 2024-11-28
> **状态**: 已完成

风险熔断机制定义了何时必须停止更新并进行修复，分为三个等级：

---

### 5.1 黄色预警（关注级）

#### 5.1.1 触发条件

满足以下任意一条即触发黄色预警：

| 编号 | 触发条件 | 数据来源 |
|------|----------|----------|
| Y-01 | **连续3章订阅下跌>10%** | 平台后台数据 |
| Y-02 | **周均订跌幅>15%** | 平台后台数据 |
| Y-03 | **读者负面评论占比>30%** | 评论区监控 |
| Y-04 | **存稿数量<8章** | 存稿台账 |
| Y-05 | **单章错别字>10个** | 读者反馈/自检 |
| Y-06 | **连续2章无爽点** | 内容自检 |
| Y-07 | **设定小矛盾被读者指出** | 评论区监控 |

#### 5.1.2 应对动作

```
触发黄色预警
      ↓
① 召开专家会诊（PM + CA + QA）
      ↓
② 分析问题根源
      ↓
③ 制定2周内改进计划
      ↓
④ 加强监控频率（每日→每章）
      ↓
⑤ 2周后评估是否解除预警
```

**具体措施**：

| 触发条件 | 对应措施 |
|----------|----------|
| 订阅下跌 | 分析近期章节爽点密度，调整后续节奏 |
| 负面评论多 | 收集具体反馈，针对性改进 |
| 存稿告急 | 减少外部事务，集中精力补稿 |
| 错别字多 | 加强自检流程，考虑增加校对工具 |
| 无爽点 | 调整大纲，确保后续章节有明确爽点 |
| 设定矛盾 | 核查Project-Bible，修正并更新设定库 |

#### 5.1.3 解除条件

- 连续7天各项指标恢复正常
- 改进措施已落地执行
- 专家会诊确认可解除

---

### 5.2 红色预警（紧急级）

#### 5.2.1 触发条件

满足以下任意一条即触发红色预警：

| 编号 | 触发条件 | 数据来源 |
|------|----------|----------|
| R-01 | **出现重大设定矛盾**（力量体系崩溃、人物严重OOC） | 读者反馈/自检 |
| R-02 | **周均订暴跌>30%** | 平台后台数据 |
| R-03 | **黄色预警2周未解除** | 预警台账 |
| R-04 | **存稿数量<3章** | 存稿台账 |
| R-05 | **章节被平台锁定** | 平台通知 |
| R-06 | **连续5章读者评分<6分** | 平台评分系统 |
| R-07 | **核心读者大规模弃书** | 追读数据 |

#### 5.2.2 应对动作

```
触发红色预警
      ↓
① 【立即】暂停日更（改为隔日更新或暂更1-3天）
      ↓
② 【24小时内】召开紧急专家会诊
      ↓
③ 【48小时内】完成问题诊断报告
      ↓
④ 【72小时内】制定紧急修复方案
      ↓
⑤ 【执行】修复+恢复更新
      ↓
⑥ 【跟踪】持续监控2周
```

**具体措施**：

| 触发条件 | 对应措施 |
|----------|----------|
| 重大设定矛盾 | 暂停更新，修正已发布章节，发布设定勘误说明 |
| 订阅暴跌 | 分析流失原因，考虑剧情大转折或重大福利 |
| 黄色预警升级 | 执行更激进的改进措施 |
| 存稿告急 | 暂停外部一切活动，全力补稿 |
| 章节被锁 | 立即修改敏感内容，申请解锁 |
| 评分过低 | 分析差评原因，调整内容方向 |
| 读者弃书 | 发布作者说明，承诺改进，加更补偿 |

#### 5.2.3 解除条件

- 问题根源已解决
- 连续14天指标稳定
- 读者反馈明显改善
- 专家会诊一致同意解除

---

### 5.3 黑色预警（危机级）

#### 5.3.1 触发条件

满足以下任意一条即触发黑色预警：

| 编号 | 触发条件 | 数据来源 |
|------|----------|----------|
| B-01 | **触碰平台红线**（涉政、涉黄、涉暴等） | 平台通知 |
| B-02 | **作品被下架/封禁** | 平台通知 |
| B-03 | **红色预警2周未解除** | 预警台账 |
| B-04 | **发生重大舆论危机**（被大规模批评/抵制） | 社交媒体监控 |
| B-05 | **核心设定需要推翻重来** | 专家会诊决议 |
| B-06 | **月均订跌幅>50%** | 平台后台数据 |

#### 5.3.2 应对动作

```
触发黑色预警
      ↓
① 【立即】全面停更
      ↓
② 【立即】发布公告告知读者
      ↓
③ 【24小时内】召开全体专家紧急会议
      ↓
④ 【72小时内】完成全文审查报告
      ↓
⑤ 【1周内】制定重建/恢复方案
      ↓
⑥ 【用户确认】是否继续项目
      ↓
⑦ 【执行】大修或终止
```

**具体措施**：

| 触发条件 | 对应措施 |
|----------|----------|
| 触碰平台红线 | 全面自查全文，删除/修改所有敏感内容 |
| 作品被下架 | 联系编辑申诉，准备整改方案 |
| 红色预警升级 | 考虑是否需要重大剧情调整或设定修正 |
| 舆论危机 | 发布官方回应，必要时道歉 |
| 核心设定推翻 | 评估返工成本，决定是否继续 |
| 订阅暴跌 | 评估项目可行性，考虑是否转型或终止 |

#### 5.3.3 解除条件

- 平台审核通过
- 整改措施全部落地
- 连续30天运营正常
- 用户明确同意恢复
- 专家全体一致同意解除

### 5.4 预警升级与降级机制

```
正常状态
    ↓ 触发黄色条件
黄色预警（关注）
    ↓ 2周未解除 或 触发红色条件
红色预警（紧急）
    ↓ 2周未解除 或 触发黑色条件
黑色预警（危机）
```

**降级路径**：

| 当前级别 | 降级条件 | 降级后级别 |
|----------|----------|------------|
| 黑色预警 | 30天稳定 + 专家全体同意 | 红色预警 |
| 红色预警 | 14天稳定 + 专家多数同意 | 黄色预警 |
| 黄色预警 | 7天稳定 + 指标恢复 | 正常状态 |

### 5.6 熔断机制执行原则

1. **快速响应**：发现问题后24小时内必须启动对应预警流程
2. **如实记录**：所有预警事件必须记录在台账中
3. **闭环管理**：每个预警必须有明确的解除条件和解除记录
4. **持续优化**：每次预警解除后复盘，优化预防措施
5. **用户知情**：红色及以上预警必须通知用户
